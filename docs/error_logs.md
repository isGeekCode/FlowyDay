# Error Logs — FlowyDay MVP

> 개발 중 발생한 에러, 실패, 예상치 못한 문제들을 기록하는 문서입니다.
> 각 이슈는 "기획 → 실제 → 원인 → 해결 → 교훈" 순서로 작성합니다.

---

## 작성 규칙

각 에러/이슈는 다음 형식으로 작성합니다:

```markdown
## YYYY-MM-DD — 에러/이슈 제목

### 1. 기획 단계의 가정
- 당시 어떻게 생각하고 있었는지

### 2. 실제 발생한 문제
- 어떤 상황에서 어떤 문제가 발생했는지

### 3. 원인 분석
- Root Cause 파악

### 4. 해결 방법
- 어떻게 해결했는지

### 5. Tech Spec/TODO에 반영해야 할 내용
- 다음 기획에서 주의할 점

### 6. 관련 커밋/PR 링크 (선택)
- 해결 커밋 링크
```
